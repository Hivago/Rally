## 2. Where did AI go wrong?

- [x] **Other: Called SaveChangesAsync after ExecuteDeleteAsync** ŌĆö `ExecuteDeleteAsync` bypasses the change tracker and hits the DB directly. Calling `SaveChangesAsync` afterwards is unnecessary and misleading.

- [x] **Other: Missing schema qualifier in raw SQL migration** ŌĆö Raw SQL in `migrationBuilder.Sql(...)` defaulted to `public` schema instead of the module's schema (e.g. `users`, `orders`). Always qualify table names with schema.

- [x] **Other: JsonStringEnumConverter only added to one JSON option chain** ŌĆö `Program.cs` has two JSON configuration points (`ConfigureHttpJsonOptions` and `AddJsonOptions`). Adding the converter to only one caused enums to serialize as integers in some responses.

**Pattern: EF Core bulk methods (ExecuteDeleteAsync/ExecuteUpdateAsync) and raw SQL need extra care ŌĆö they bypass the normal EF pipeline and require explicit schema awareness.**
